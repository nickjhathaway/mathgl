After several years of MathGL developing I have some suggestions which will make the MathGL interface incompatible with one of versions 1.*. So, I plan to prepare MathGL v.2.0 which will be not absolutely compatible with v.1.*. Most of differences is connected with MGL scripts, utility variables and functions (remove ones marked as obsolete in documentation) and some plotting functions with goal to hide and unify extra arguments. I'll try to keep interface as compatible as possible with v.1.* but ... By the way I'll prepare one more version (v.1.11) with interface compatible with v.1.* before release of v.2.0.

So, below I put suggested changes for v.2.0. I divide it by 2 groups: planned and already included in SVN. Red colors denote incompatible changes. So, if you have any suggestions, improvements or bugs than e-mail me and I'll include it into the "wish" list and try to add to the new release.

Planned improvements (wish list ☺):
  • Colorbar: print more digits for manual arrays ... but avoid overlapping.
  • Miss MAJOR labels or decrease its fontsize if labels too close.
  • Allow rotate labels if they too dense.
  • Check that enough digits is printed to distinguish the labels.
  • Allow "time"-labels.
  • Contour label should not rotated on ~180 degree
  • Save glyph then it is used for mglGraphPS -- allow different font family in EPS|SVG
  • Import/Export via GIF (as 3d data) ??? + 16bit PNG
  • Add option ':[xyzd]' in color scheme for coloring along coordinate (default 'z') -- not sure about it.
  • Save ALL plot styles for each curve (0...N) for each subplot (0...M) in format \leg{N_M}
  • Add TeX command \leg{plot style} for line+mark of specified style.
  • Export to X3D
  • Quaternary diagrams plotted in a tetrahedron: equivalent to ternary diagrams, but in 3D.
  • Legend in OpenGL mode
  • Multithreading by "subplots" -- for parser first of all
  • Swap color and font style in text option ('bR:r' -> 'r:bR')
  • Hide 'dir' argument as ':[xyz]' for cont3, dens3, contf3, grid3 + remove conta, densa, grida, contfa.
  • Allow slice definition as real: sl<1 ? sl=int(n*sl) : int(sl);
  • Tune colors in {} -- for example, 'b{3}' is old 'b3'
  • Hide unused arguments (like "size")
  • Hide/remove obsolete functions/variables/commands.
